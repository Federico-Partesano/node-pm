# Sessions Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ship `pm session create|list|show|run|remove` (CLI + TUI page) — a named collection of long-running shell commands per project, spawnable in one keystroke, with live log streaming in a multi-pane Ink view.

**Architecture:** A pure-data `Session` in the manifest. A `SessionRunner` orchestrator spawns one `execa` child per terminal, multiplexes their stdout/stderr into a typed `SessionEvent` async generator, and exposes `kill(name)` / `restart(name)`. CLI renders prefixed lines; TUI renders N inline `LogTail` panes.

**Tech Stack:** TypeScript (ESM), Node 20+, vitest, ink, commander, `execa`, `zod`.

**Reference spec:** `docs/superpowers/specs/2026-05-21-sessions-design.md`.

## Task 3: RingBuffer + SessionRunner

**Files:**
- Add: `src/core/session-runner.ts`
- Add: `test/core/session-runner.test.ts`

- [ ] **Step 1: `RingBuffer<T>`** — `push`, `toArray`, capacity check. Tiny standalone class in the same file.

- [ ] **Step 2: `SessionEvent` union** as in the spec.

- [ ] **Step 3: `SessionRunner` class**:
  - Constructor takes `{ resolveProjectPath: (ref: string) => string }`.
  - `async *run(session, { signal })` async generator.
  - For each `TerminalSpec`: spawn via `execa(cmd, { shell: true, cwd, stdio: 'pipe', env: { ...process.env, ...spec.env }, signal, reject: false })`.
  - Attach `readline` on `child.stdout` and `child.stderr`.
  - Push events into an internal `queue: SessionEvent[]` with a `resolveNext` continuation, same pattern as `PackageManager.install`.
  - Track `Map<name, child>` for `kill(name)` / `restart(name)`.
  - `kill(name)`: send `SIGTERM`, wait 3s, then `SIGKILL`. Emit `killed`.
  - `restart(name)`: kill if running, respawn from the original `TerminalSpec`, replace child in map.
  - Emit `all-done` when every child has exited.

- [ ] **Step 4: Tests**:
  - Single terminal `echo hi` → `start`, `line('hi','stdout')`, `exit(0)`, `all-done`.
  - Single terminal `node -e "process.stderr.write('boom\\n'); process.exit(1)"` → `line('boom','stderr')`, `exit(1)`, `all-done`.
  - Two terminals, kill one mid-flight → first emits `killed`, second still runs, `all-done` after second exits.
  - Spawn with a non-existent binary → `error` event, no throw, runner still callable.
  - RingBuffer capacity (push 600, assert length 500).
  - AbortSignal triggers SIGTERM on all children (use a 200ms sleep terminal, abort, assert `exit` arrives within 4s).

- [ ] **Step 5: Commit** `feat(core): SessionRunner with multi-terminal spawn`.
